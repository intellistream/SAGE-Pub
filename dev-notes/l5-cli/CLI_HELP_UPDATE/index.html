    Date: 2025-10-26\ Author: SAGE Development Team\ Summary: SAGE CLI 帮助信息更新总结 - 对齐 sage (packages/sage-cli) 与 sage-dev (packages/sage-tools) 的帮助文本

    SAGE CLI 帮助信息更新总结¶

    🎯 更新目标¶

    确保 sage --help 和 sage-dev --help 能够清晰展示最新的命令组织结构。

    ✅ 已完成的更新¶

    1. 主 CLI 帮助信息 (sage --help)¶

    文件: packages/sage-cli/src/sage/cli/main.py

    更新内容¶

    1. Platform / Apps 分组说明更清晰
    2. Typer 应用帮助字符串按照 “Platform” 与 “Apps” 两大类描述命令
    3. 为每个类别补充 emoji + 典型操作，便于在 --help 输出中快速定位目标命令
    4. 显式提示开发命令迁移到 sage-dev
    5. 主帮助文本直接告诉用户：平台/应用命令通过 sage 调用，开发工具通过 sage-dev
    6. 示例命令覆盖 sage cluster ... 与 sage-dev quality ... 两类用法
    app = typer.Typer(
    name="sage",
    help="""🚀 SAGE - Streaming Applied to GEneral data stream

    命令分类：
    • Platform  - 集群管理和作业调度
    • Apps      - 应用层服务（LLM、Chat等）

    开发工具：
      开发命令请使用 sage-dev (由 sage-tools 包提供)
""",
    no_args_is_help=True,
)

    🎨 设计特点¶

    1. 清晰的视觉分层¶

    • sage 输出按 Platform / Apps 划分命令
    • sage-dev 输出聚合 9 个开发命令组
    • Emoji + Rich 布局突出不同职责

    2. 渐进式信息披露¶

    • sage --help: 列出平台/应用命令，并提醒开发任务请使用 sage-dev
    • sage-dev: 显示欢迎信息和快速开始示例
    • sage-dev --help: 显示所有命令组
    • sage-dev <group> --help: 显示特定组的命令

    3. 一致的帮助格式¶

    • 所有命令都有 emoji 前缀
    • 简短描述 + 详细说明
    • 示例和使用场景

    💡 用户体验¶

    发现性¶

    用户可以通过层级式的帮助系统逐步发现功能：

    1. sage --help → 看到 Platform / Apps 命令并获知开发命令位置
    2. sage-dev → 看到欢迎信息和快速开始
    3. sage-dev --help → 看到所有命令组
    4. sage-dev quality --help → 看到质量检查的所有命令

    学习曲线¶

    • 新用户: 通过欢迎信息和示例快速上手
    • 熟悉用户: 使用 --help 快速查找命令
    • 高级用户: 直接使用命令，无需查看帮助
